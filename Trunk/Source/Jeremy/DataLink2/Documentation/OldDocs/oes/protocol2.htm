Yes, if byte 5 is blanked (0x3A) then we are in last minute of play (Minutes = 0) and bytes 2-3 will be seconds and byte 4 will be tenths of seconds.

 

The standard message that I sent you is one direction – controller to scoreboard, during the game there is no response message from the scoreboard to controller.  The remainder of communication is transmitted in both directions between the controller and scoreboard.  The following are the other types of messages that you will be seeing, but ignoring:

 

- Initial Setup

            - When the controller is turned on, it requests information from the scoreboard

            - The message from the controller to the scoreboard:

                        - STX (0x02)

                        - “r”

                        - Checksum

                        - CR (0x0D)

            - The response from the scoreboard to the controller is the same as the message that I sent to you earlier (Standard game message) except the “t” for BYTE 1 is a “r”

 

- Team Names

            - There are a few messages that go back and forth regarding the team names, they all start with numerous combinations (5-20 repeats) of 0x5D, 0x20.

 

I hope this helps, please let me know if I can be of any further assistance.

Attached are the protocols for hockey & basketball for San Antonio and below is running through the basketball for an example.  To start with, this protocol may seem a little confusing, this is because our scoreboard is a dummy terminal meaning the protocol has to have everything in it to control everything on it.  Please feel free to contact me on any of this.

 

BASKETBALL Protocol

 

1.       9600 Baud, 8N1

2.       Both RS232 & RS485 are available.

3.       The messages are sent every 1/10th of a second

4.       Values are generally sent in ASCII coded format, except as noted

5.       Any values that need to be blank we send a 0x3A instead of 0x30

 

BYTE                Character         Value   Note

0                      stx                    0x02

 

1                      't'                      0x74     Scoreboard Message Identify

 

2-5                    mmss               ASCII*  GAME TIME (Minutes, Seconds OR Seconds, Tenths in last minute of play)

            Note:

1.       The second digit for minutes contains the upper colon information.  If the colon needs to be on, take the ASCII value and add 0x80.

2.       The first digit for seconds contains the lower colon information. If the part of the colon needs to be on, take the ASCII value and add 0x80.

1.       Eg. Time = 12:34, message = 0x31, 0xB2 (For '2' + 0x80 for upper colon on), 0xB3 (For '3' + 0x80 for lower colon on), 0x34

2.       Eg. Time (In last minute) = 59.2, message = 0x35, 0x39, 0xB2 (For '2' plus 0x80 for lower colon on), 0x3A (For blank)

 

6                      x                      ASCII    PERIOD

 

7-8                    xx                     ASCII*  HOME SCORE

9-10                  xx                     ASCII*  AWAY SCORE

            Note: If the score is greater than 99, 0x80 is added to both of the other ASCII values for that score

3.       Eg. Home Score = 98, message = 0x39, 0x38

4.       Away Score = 102, message = 0xB0, 0xB2

 

11                     x                      ASCII*  HOME TIME OUTS LEFT

            Note: If team has 20 second time out left, 0x80 is add

5.       Eg. TOL = 2, has 20 second timeout left, message = 0xB2

 

12                     x                      ASCII*  AWAY TIME OUTS LEFT

            Note: Same as above for Home TOL

 

13                     x                      ASCII    HOME TEAM FOULS

 

14                     x                      ASCII    AWAY TEAM FOULS

 

15                     'S'                    0x53     Bonus/Possession Message Idenifty

 

16                     x                      ASCII*  HOME POSS/BONUS

            Note: Start with 0x30, add :

6.       0x01 if Home has possession

7.       0x02 if Home has bonus

 

17                     x                      ASCII*  AWAY POSS/BONUS

            Note: Start with 0x30, add :

8.       0x01 if Away has possession

9.       0x02 if Away has bonus

 

18-19                xx                     ASCII    SHOT CLOCK

 

20                     x                      ASCII*  OUTPUTS

            Note: Start with 0x30, add:

10.   0x01 for game time horn

11.   0x02 for shot clock horn

12.   0x04 for end of quarter light

13.   0x08 for ad panels - always on when standard protocol is sent

 

21                     x                      ASCII    CHECKSUM

            Note: OES Checksum

14.   Use a 1 Byte Accumulator

15.   Add up all bytes from STX up to and including OUTPUTS

16.   AND with 0x7F

17.   If result less than 0x20, add 0x20

 

22                     ETX                  0x0D    

 

 

As I mentioned earlier, this is a little different of a protocol.  We do this to minimize the processing on the scoreboard end.  If you have any questions on this please don't hesitate to call.

 

On last thing, our controller also sends out other messages that are used to put team names on the scoreboards.  The format of these messages are completely different from that above.  Do you need this message protocol to be able to ignore these messages.
